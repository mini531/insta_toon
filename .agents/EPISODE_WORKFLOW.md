# 인스타툰 에피소드 제작 루틴 (지침)

새 에피소드를 기획하고 스토리보드까지 만드는 표준 절차다. ep01 작업을 그대로 재현할 수 있게 정리했다.
캐릭터 설정은 `.agents/AGENTS.md`, 화풍/구성/포맷 기준은 이 문서를 따른다.

---

## 1. (선택) 벤치마킹

참고할 인스타툰 게시물이 있으면 먼저 캡처/분석한다.
- `benchmark/<게시물코드>/` 폴더 생성.
- 비공개/로그인 월이면 Playwright + 로그인 세션으로 캐러셀을 한 장씩 캡처(`benchmark/crawl_login.js`, `capture_all.js` 재사용). 자격증명은 `benchmark/.ig_login`(gitignore)으로 전달.
- `ANALYSIS.md`에 그림체/컷 구성/텍스트 처리/비율을 정리. 특히 "한 컷에 몇 장면이 들어가는지"를 본다.

---

## 2. 시나리오 기획

### 2-1. 기본 골격
- 분량: 슬라이드 10장. 마지막 컷은 다음 화로 이어지는 클리프행어로 끝낸다.
- 흐름 예: 후킹(표지) -> 도입 -> 배경/사정 -> 일상의 갈등(2~3컷) -> 감정 고조 -> 클리프행어.

### 2-2. 한 컷 = 2~3 장면 (핵심)
- 벤치마크처럼 한 슬라이드 안에 장면을 위아래로 2~3개 적층한다(밋밋한 1컷 1장면 금지).
- 장면 분리는 배경 색 띠로. 적층은 상/하 배치라 뒤의 4:5 좌우 크롭에 안전하다.
- 전경/배경 동시 표현도 활용(예: 전경의 압박 + 배경의 방관).

### 2-3. 타깃 공감 양념
- 타깃 독자가 "내 얘기다" 싶은 디테일을 구체 사물/상황으로 넣는다.
- ep01 예시: 난임 자가주사와 동그라미 친 달력, 명절 질문공세, "누구네 며느리는 벌써 둘", 보약 강요, 몸 훑기 + 남편의 방관, 친정 비난, 죄책감의 밤.
- 추상적 서술 대신 장면으로 보여준다.

### 2-4. 텍스트
- 나레이션: 상/하단 가로 띠. 대사: 화자색 말풍선.
- 그림과 분리해 후합성한다(그림에는 글자를 넣지 않는다).

## 3. 새 ep 폴더 생성 + 백업

```
mkdir epNN
# 같은 화를 다시 그리는 경우에만: 기존 이미지를 백업
mkdir epNN/backup_v1 && cp epNN/01.png ... epNN/backup_v1/   # md5로 동일성 확인
```
- 신규 화는 backup 불필요. 재생성 화는 반드시 backup 먼저(덮어쓰기 안전장치).

## 4. 스토리보드 작성

1. `_template/storyboard.template.html` 를 `epNN/storyboard.html` 로 복제.
2. 파일 상단 `<script>` 의 **CONFIG** 와 **SLIDES** 만 편집한다(CSS/렌더/배치 로직은 손대지 않는다).
   - `CONFIG.ep` = 에피소드 번호(예: '02'), `CONFIG.title`/`subtitle`, `CONFIG.hasPrev`(재생성 시 true).
   - `SLIDES` = 10개 슬라이드 데이터. 스키마는 부록 A 참조.
3. 저장 후 브라우저로 열어 확인. 하단 "일괄 생성 프롬프트" 블록과 컷별 저장 경로가 자동 생성된다.
4. `epNN/prompts.md` 는 스토리보드 하단 일괄 블록을 복사해 만들거나, ep01/prompts.md 형식을 따른다.

---

## 5. 제작 표준 (체크리스트)

- [ ] 비율: 최종 4:5 세로. 생성은 1:1만 되므로 **생성 후 좌우 센터 크롭으로 4:5 변환**. 핵심 요소는 중앙 4:5 안전영역에, 좌우는 여백.
- [ ] 화풍: 플랫 2D 카툰, 저채도 파스텔, 가는 갈색 라인, 음영 최소. 반실사/수채 금지. 10컷 화풍 1종 통일.
- [ ] 캐릭터: `character_sheet.png` 를 참조 이미지로 첨부. 헤어/의상색/얼굴 동일 유지.
- [ ] 두 노인 구분: 시아버지 = 브라운 카라 스웨터 / 친정아빠 = 회갈색 카라셔츠 위 스웨터.
- [ ] 텍스트: 그림에 글자 없이 여백만. 빈 말풍선이 아니라 합성용 빈 공간.
- [ ] 워터마크: '@toon_observer_' 를 이미지마다 임의 위치 2곳에 옅고 흐리게.
- [ ] 저장 지시: 각 프롬프트에 `C:\project\insta_toon\epNN\NN.png` 명시.
- [ ] 한 컷 2~3 장면 적층 적용.

---

## 6. 검증

- Playwright(헤드리스)로 `storyboard.html` 을 열어 컷 수(10), 이미지 깨짐, 콘솔 에러, 일괄 블록(공통 + 10컷 + 저장줄 10개)을 확인한다.
- 다중컷/명절 등 복잡한 슬라이드는 카드 스크린샷으로 레이아웃을 눈으로 본다.

---

## 7. 이미지 생성 -> 크롭 -> 합성

1. 스토리보드 하단 [전체 프롬프트 복사] -> 이미지 생성기에 전달(캐릭터 시트도 함께 첨부).
2. 1:1 결과를 `epNN/01.png ~ 10.png` 로 저장.
3. 좌우를 잘라 4:5로 센터 크롭.
4. 스토리보드 레이아웃대로 나레이션 띠/말풍선 텍스트를 합성.

## 부록 A. SLIDES 스키마

```js
{
  num:'01', file:'01.png', title:'컷 제목',
  titleZone:'표지 제목',          // (선택) 표지 컷에만
  narrTop:'상단 나레이션',         // (선택)
  narrBottom:'하단 나레이션',      // (선택)
  narrBottomDark:'하단 나레이션(어두운 띠)', // (선택, 클리프행어 등)
  scenes:[                        // 1~3개. 위->아래 순서로 적층
    { label:'장면 묘사',
      bub:{ cls:'mil', who:'화자', text:'대사' } }  // (선택) 말풍선
  ],
  dialogue:[ {who:'화자', text:'"대사"'} ],  // (선택) 대사 목록
  cast:['등장인물'],
  prompt:'이미지 생성 프롬프트(공통 블록과 합쳐 사용)',
  stack:'2장면 적층 메모'          // (선택)
}
```

| husband | 남편 |
| mil | 시어머니 |
| fil | 시아버지 |
| wmom | 친정엄마 |
